# INFRA
- flutter desktop
- sqlite para armazenamento de dados
- o sistema deve possuir um formulário de login simples. as informações necessárias para autenticação via endpoint serão vinculadas ao usuário diretamente no banco de dados
- existirão 4 telas, uma para cada módulo
- todo cadastro deve ter um registro pai que será o edital, tudo se deriva do edital
- para envio das informações aos endpoints é necessário fazer login via endpoint com as informações salvas no banco, o sistema de ter uma confirmação no momento do envio de alguma fase, inclusive informando o usuário logado no momento
- todo retorno do endpoint deve ser armazenado em uma tabela de logs
